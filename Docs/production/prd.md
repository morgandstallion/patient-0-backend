# Local Pickup E-Commerce Platform

## Overview

This project is an e-commerce platform that allows customers to purchase fashion accessories and low-cost products online and either receive delivery or collect their orders from a designated pickup location.

The primary goal is to provide a convenient local shopping experience while reducing the waiting time associated with international shipping.

This project is also intended as a learning platform for backend development, system design, authentication, order management, inventory management, and logistics workflows.

---

# Problem Statement

Many customers want affordable products but face challenges such as:

- Long international shipping times
- Uncertainty about package arrival
- Lack of convenient local pickup options
- Difficulty tracking order fulfillment

This platform aims to solve these problems by maintaining local inventory and providing structured delivery and pickup processes.

# User Roles

## Customer

A customer can:

- Create an account
- Browse products
- Add products to cart
- Place orders
- Pay for orders
- Track order status
- Schedule pickups
- Verify identity during pickup

## Admin

An admin can:

- Add products
- Edit products
- Remove products
- Manage inventory
- View all orders
- Update order status
- Manage pickup schedules
- View analytics

## Pickup Staff (Future)

Pickup staff can:

- Scan customer QR codes
- Verify customer identity
- Release orders
- Record completed pickups

# Version 1 Scope

The first version of the system includes:

- User registration and login
- Product catalog
- Product details page
- Shopping cart
- Checkout
- Order creation
- Order history
- Admin dashboard
- Product management
- Order management

Version 1 excludes:

- QR verification
- OTP verification
- Pickup scheduling
- Delivery management
- Analytics
- Staff accounts

# Success Criteria

A customer can:

1. Create an account.
2. Browse products.
3. Add items to cart.
4. Complete checkout.
5. View their order.

An admin can:

1. Add products.
2. View products.
3. Manage inventory.
4. View customer orders.
5. Update order status.

If these actions work correctly, Version 1 is considered successful.
